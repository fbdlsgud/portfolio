# Ultrawork Phase Gates

## PLAN Gate

- Requirements are mapped to tasks.
- Assumptions are explicit.
- Non-goals are explicit.
- Validation commands are known.
- Risky areas are identified.

## IMPL Gate

- Implementation matches the approved scope.
- Files changed are explainable.
- Tests or validation were attempted.
- No unrelated cleanup was mixed in.

## VERIFY Gate

- QA findings are verified.
- No critical/high issue remains unresolved.
- No obvious regression is present.
- Accessibility and security were considered when relevant.

## REFINE Gate

- Fixes target root causes.
- Revalidation was performed.
- Deferred items are documented.

## SHIP Gate

- Feature log exists for meaningful work.
- Error/review cases exist when useful.
- Proposed rules are separate from approved permanent rules.
- Final report is concise and evidence-backed.
