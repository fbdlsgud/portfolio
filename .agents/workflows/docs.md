# Docs Workflow

Use this when updating feature logs, implementation notes, decisions, or docs.

## Steps

1. Determine which docs are affected.
2. Read current docs before editing.
3. Update only relevant sections.
4. Keep docs concise and searchable.
5. Link to source files when helpful.
6. Avoid changing source code.

## Standard Locations

- `docs/ai/conventions.md`
- `docs/ai/feature-log/`
- `docs/ai/error-cases/`
- `docs/ai/review-cases/`
- `docs/ai/decisions/`
- `docs/ai/proposed-rules/`
