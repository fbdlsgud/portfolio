# Product Discovery Workflow

Use when the user gives a rough app idea, stack, or feature list.

## Steps

1. Identify app type and target users.
2. List confirmed requirements.
3. Suggest missing MVP features.
4. Split suggestions into MVP and later.
5. Ask only truly blocking questions.
6. Update `docs/ai/conventions.md`.
7. Create implementation plan if the user wants to proceed.

## Output

```text
Confirmed:
Suggested MVP:
Later:
Open questions:
Recommended next implementation slice:
```
