# Architecture Workflow

Use for structural decisions, module boundaries, service boundaries, data
ownership, and ADRs.

## Steps

1. Read project context.
2. Frame the decision.
3. Identify constraints and non-goals.
4. Compare at least two options for significant decisions.
5. Recommend one option with tradeoffs.
6. Record durable decisions in `docs/ai/decisions/`.

## Output

```text
Problem:
Options:
Recommendation:
Risks:
Validation:
Decision record:
```
