# Plan Workflow

Use this before implementation when the request is broad, unclear, or spans
multiple areas.

## Steps

1. Read project context.
2. Restate the goal.
3. Identify constraints and non-goals.
4. Break work into tasks.
5. Assign likely subagents.
6. Define acceptance criteria.
7. Stop before implementation unless the user asks to proceed.

## Output

Use:

```text
Goal:
Scope:
Non-goals:
Tasks:
Acceptance criteria:
Suggested agents:
Validation:
```
