# Frontend Rules

- Inspect existing component patterns before creating new ones.
- Prefer the project's existing UI library and design tokens.
- Keep UI state local unless shared state is clearly needed.
- Use server-state tools for remote data when the project has them.
- Validate loading, empty, error, and success states.
- Preserve keyboard accessibility for interactive elements.
- For visible UI changes, verify with browser or screenshots when practical.
- Record meaningful UI work in `docs/ai/feature-log/`.
