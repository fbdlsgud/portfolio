# Architecture Skill

Use for structural decisions, module boundaries, data ownership, tradeoffs, and
ADR-style decisions.

## Process

1. Frame the decision.
2. Read project context and relevant code.
3. Identify constraints and quality attributes.
4. Compare at least two meaningful options for significant decisions.
5. Recommend one option with risks and validation steps.
6. Record durable decisions in `docs/ai/decisions/`.
