# QA Skill

Use for security, correctness, performance, accessibility, maintainability, and
test coverage review.

## Process

1. Define review scope.
2. Read project context and changed files.
3. Verify every finding.
4. Prioritize security and correctness.
5. Report file-specific findings with remediation.
6. Record reusable review patterns in `docs/ai/review-cases/`.
