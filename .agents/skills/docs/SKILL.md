# Docs Skill

Use for project context, feature logs, error cases, review cases, decisions, and
proposed rules.

## Process

1. Determine the correct docs/ai target.
2. Read existing docs before editing.
3. Keep records concise and searchable.
4. Include concrete file paths and commands when useful.
5. Do not include secrets.
6. Do not change source code.
