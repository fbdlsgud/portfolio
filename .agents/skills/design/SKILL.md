# Design Skill

Use for product UX, interaction design, visual hierarchy, accessibility, and
design QA before implementation.

## Process

1. Read project context and existing UI conventions.
2. Identify target users, primary tasks, and constraints.
3. Define information architecture, states, and interaction behavior.
4. Check accessibility, responsive behavior, and empty/loading/error states.
5. Coordinate with frontend or mobile implementation agents.
6. Record durable design decisions in `docs/ai/decisions/` when useful.
