# Difficulty Guide

## Tiny

One-file change or direct answer.

Use direct implementation.

## Small

Focused feature or bug.

Use implement or debug workflow.

## Medium

Multiple files or unclear edge cases.

Use work or orchestrate.

## Large

Cross-domain, risky, or release-like work.

Use ultrawork.

## Persistent

The user wants repeat-until-done with mechanical criteria.

Use ralph.
