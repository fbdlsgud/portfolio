# Quality Principles

## 1. Correctness Before Cleverness

The implementation must satisfy the requested behavior and avoid regressions.

## 2. Smallest Safe Change

Prefer focused edits over broad rewrites.

## 3. Evidence Over Assumption

Use tests, builds, file checks, browser/device checks, logs, or concrete code
inspection as evidence.

## 4. Durable Learning

Record repeated lessons, but promote permanent rules only after review.
