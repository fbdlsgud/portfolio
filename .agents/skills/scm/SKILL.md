# SCM Skill

Use for git status, staging, commits, branches, changelog, and PR summaries.

## Process

1. Inspect `git status --short`.
2. Identify task-owned changes.
3. Avoid staging unrelated user changes.
4. Run requested validation before commit.
5. Write concise commit/PR text.
6. Never run destructive git commands without explicit approval.
