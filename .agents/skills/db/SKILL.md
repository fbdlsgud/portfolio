# Database Skill

Use for schemas, migrations, indexes, transactions, Firestore collections, SQL,
and data integrity.

## Process

1. Inspect existing schema/data model.
2. Define ownership and access patterns.
3. Add constraints or validation where appropriate.
4. Consider indexes and query costs.
5. Avoid destructive changes without explicit approval.
6. Coordinate with backend/mobile/frontend consumers.
